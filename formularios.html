Diseñando tu formulario

En esta sección, crearemos un formulario de contacto simple. Hagamos un bosquejo aproximado.

Nuestro formulario contendrá tres campos de texto y un botón. Le pedimos al usuario su nombre, su correo electrónico y el mensaje que desea enviar. Al presionar el botón se enviarán sus datos a un servidor web.

Implementando nuestro formulario HTML

Ok, ahora estamos listos para ir a HTML y codificar nuestro formulario. Para crear nuestro formulario de contacto, utilizaremos los siguientes elementos HTML: <form>, <label>, <input>, <textarea> y <buttom>.

Antes de continuar, cree una estructura básica de Html, donde ingrésaremos el formulario.

El elemento <form>

Todos los formularios HTML comienzan con un elemento <form> como este:

Este elemento define formalmente una forma. Es un elemento contenedor como un elemento <div> o <p> , pero también admite algunos atributos específicos para configurar la forma en que se comporta el formulario. Todos sus atributos son opcionales, pero se considera una práctica recomendada establecer siempre al menos el atributo de action y el atributo de method .

  • El atributo de action define la ubicación (URL) donde se deben enviar los datos recopilados del formulario cuando se envían.
  • El atributo del method define con qué método HTTP enviar los datos (puede ser "obtener" o "publicar").

Por ahora, agregue el elemento <form> en su cuerpo HTML.

El elemento <button>

Nuestra forma está casi lista; solo tenemos que agregar un botón para permitir que el usuario envíe sus datos una vez que haya completado el formulario. Esto se hace simplemente usando el elemento <button> ; agregue lo siguiente justo arriba de la etiqueta de cierre </form>:

Verá que el elemento <button> también acepta un atributo de type ; esto acepta uno de los tres valores: submit, reset o button.

  • Un clic en un botón de submit (el valor predeterminado) envía los datos del formulario a la página web definida por el atributo de action del elemento </form>.
  • Un clic en un botón de reset restablece todos los widgets de formulario a su valor predeterminado inmediatamente. Desde el punto de vista de UX, esto se considera una mala práctica.
  • Un clic en un botón no hace ... ¡nada! Eso suena tonto, pero es increíblemente útil para crear botones personalizados con JavaScript.
